# Лабораторная работа: Kafka 4.0 KRaft — Одноброкерный кластер

Эта лабораторная работа разворачивает одноброкерный кластер Apache Kafka 4.0 в режиме KRaft
(Kafka Raft) без ZooKeeper. Режим KRaft является единственным поддерживаемым режимом в Kafka 4.0+.

Лаборатория содержит три упражнения, охватывающих Модули 01-04 курса:
- Упражнение 01: Основы Producer
- Упражнение 02: Consumer Groups
- Упражнение 03: Инспекция лог-сегментов

---

## Требования

- Docker версии 24.0 или новее
- Docker Compose версии 2.20 или новее

Проверьте версии командами:

```
docker --version
docker compose version
```

---

## Запуск лаборатории

Перейдите в директорию лабораторной работы:

```
cd content/courses/kafka-course/labs/core-architecture
```

Запустите контейнеры в фоновом режиме:

```
docker compose up -d
```

Kafka запускается в режиме KRaft. Контейнер `lab` дожидается готовности Kafka через
healthcheck перед стартом. Ожидаемое время запуска — 30-60 секунд.

Проверьте статус контейнеров:

```
docker compose ps
```

Оба контейнера (`kafka-lab` и `kafka-lab-python`) должны показывать статус `running`.

---

## Подключение к контейнеру упражнений

После запуска подключитесь к Python-контейнеру:

```
docker compose exec lab bash
```

Внутри контейнера будет доступна директория `/app/exercises/` с файлами упражнений.

## Упражнения

### Упражнение 01: Основы Producer

Запустите из контейнера `lab`:

```
python exercises/01_producer_basics.py
```

Ожидаемый результат: 10 строк с информацией о партиции, офсете и временной метке
для каждого отправленного сообщения. Пример:

```
Сообщение order-1 доставлено: partition=0, offset=0, timestamp=1776320400000
Сообщение order-2 доставлено: partition=1, offset=0, timestamp=1776320400001
...
```

Упражнение демонстрирует:
- Создание KafkaProducer с настройками `acks=all` и `compression_type=gzip`
- Создание топика `orders` с 3 партициями
- Отправку 10 сообщений с ключами и JSON-значениями
- Получение метаданных доставки через будущее (Future)

### Упражнение 02: Consumer Groups

Запустите из контейнера `lab`:

```
python exercises/02_consumer_groups.py
```

Ожидаемый результат: вывод назначения партиций для каждого consumer и
полученные сообщения. Упражнение показывает, как Kafka распределяет партиции
между потребителями одной группы.

Упражнение демонстрирует:
- Создание топика `events` с 4 партициями
- Двух KafkaConsumer в одной группе `analytics-group`
- Назначение партиций и получение сообщений

### Упражнение 03: Инспекция лог-сегментов

Запустите из хостовой машины (не из контейнера `lab`):

```
bash exercises/03_log_inspection.sh
```

Или выполните команды вручную внутри контейнера Kafka:

```
docker compose exec kafka bash
```

Ожидаемый результат: список топиков, информация о директориях лог-сегментов
и файлы `.log`, `.index`, `.timeindex` в директориях партиций.

Упражнение демонстрирует:
- Вывод списка топиков через `kafka-topics.sh --bootstrap-server`
- Инспекцию директорий логов через `kafka-log-dirs.sh`
- Структуру файлов лог-сегмента: `.log`, `.index`, `.timeindex`

## Решение проблем

### Порт 9092 занят

Если порт 9092 уже используется другим процессом, измените маппинг портов
в `docker-compose.yml`:

```yaml
ports:
  - "19092:9092"
```

После изменения обновите переменную `KAFKA_BOOTSTRAP_SERVERS` при запуске Python-скриптов:

```
KAFKA_BOOTSTRAP_SERVERS=localhost:19092 python exercises/01_producer_basics.py
```

### Kafka не запускается

Проверьте логи Kafka-контейнера:

```
docker compose logs kafka
```

Убедитесь, что healthcheck проходит:

```
docker compose ps
```

Если контейнер `kafka-lab` показывает `unhealthy` — подождите ещё 30 секунд
и проверьте снова. В редких случаях первый запуск занимает до 90 секунд.

### Контейнер lab не запускается

Если контейнер `kafka-lab-python` не запускается, убедитесь, что образ собран:

```
docker compose build lab
```

Затем повторите запуск:

```
docker compose up -d lab
```

### kafka-python не находит брокер

Убедитесь, что переменная окружения `KAFKA_BOOTSTRAP_SERVERS` установлена в `kafka:9092`
внутри контейнера `lab` (это значение по умолчанию из `docker-compose.yml`).
При запуске скриптов напрямую на хостовой машине используйте `localhost:9092`.

## Архитектура лаборатории

```
Контейнер kafka-lab (apache/kafka:4.0.0)
  - Режим: KRaft (broker + controller на одном узле)
  - node.id: 1
  - process.roles: broker,controller
  - Порты: 9092 (PLAINTEXT), 9093 (CONTROLLER, внутренний)
  - Данные: volume kafka-data -> /var/kafka/logs

Контейнер kafka-lab-python (python:3.12-slim)
  - kafka-python==2.0.6
  - Директория: /app/exercises/
  - KAFKA_BOOTSTRAP_SERVERS: kafka:9092
```

Брокер работает в режиме KRaft combined mode: один процесс выполняет роли и брокера,
и контроллера. Это минимальная конфигурация для разработки и обучения.
